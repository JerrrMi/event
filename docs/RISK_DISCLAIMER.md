# 风险提示与免责声明

在使用本工具前，请完整阅读以下内容。

## 工具定位

本仓库提供的 **10 分钟事件合约 ARIMA-GARCH 预测提醒工具** 仅用于：

- 根据 Binance 公开市场数据（K 线、成交量、盘口等）进行统计建模（ARIMA 方向预测 + GARCH 波动率风险校正）；
- 输出「涨 / 跌 / 观望」类 **预测提醒**；
- 在置信度达到阈值时，通过 Telegram 向用户发送通知。

ARIMA-GARCH 仍属于 **预测提醒** 范畴：GARCH 仅用于改善波动率与信噪比过滤，**不改变**「不自动下单、需人工确认」的边界。

## 明确不做的事项

本工具 **不会** 也 **不能**：

- **自动下单**：不连接、不调用任何事件合约或交易所下单接口，不代替用户完成开仓、平仓或资金操作；
- **构成投资建议**：所有输出均为基于历史与实时数据的模型推断，不代表对未来价格的承诺或推荐；
- **保证盈利**：ARIMA 为线性统计模型，GARCH 仅刻画条件波动率、不直接预测方向；二者对加密市场突发波动、消息面冲击、盘口操纵等非线性因素捕捉能力有限，历史回测表现 **不保证** 未来有效；
- **替代人工判断**：即使推送了「涨」或「跌」信号，用户仍需自行核实行情、规则、仓位与风险承受能力后再决策。

## 数据与模型局限

- 第一版以 Binance 现货/合约公开行情为输入，与具体「事件合约」产品可能存在价差、规则或结算差异；
- 回测若存在未来函数、样本过短或参数过拟合，将导致虚高的胜率评估；
- 实时推送前，应先在不少于 30 天的历史数据上完成无未来函数的滚动回测，并定期复测。

## 建议使用方式

1. 先完成历史数据下载与回测，确认信号质量可接受；
2. 使用 `DRY_RUN=true` 或 `--dry-run` 验证数据、模型与日志；
3. 通过 `python -m src.notify.telegram --test` 确认 Telegram 可达；
4. 仅在充分理解风险且愿意 **人工确认** 每一条信号后，再开启真实推送；
5. 出现异常行情、连续失效或接口变更时，**立即暂停** 实时提醒并重新回测。
